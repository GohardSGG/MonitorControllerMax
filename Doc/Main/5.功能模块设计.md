# 5. 功能模块设计

## 5.1 Protocol 层 (mcm_protocol)

定义了系统的"通用语言"。

- **`config.rs`**: `AppConfig` 结构体，序列化为 JSON，存储 OSC/Network 端口配置。
- **`web_structs.rs`**: 
    - `WebSharedState`: 使用 Atomic 字段实现的线程安全状态（端口、运行标志、本机 IP）。
    - `WebCommand`: 客户端 → 服务器的 WebSocket 命令枚举（带 `#[serde(tag = "type")]` 标签分发）。
    - `WebState` / `ChannelState`: 服务器 → 客户端的状态推送结构。
- **`osc_structs.rs`**: OSC 地址定义、`OscOutMessage` 输出消息枚举和 `ChannelLedState` 通道 LED 状态。
- **`network_structs.rs`**: Master-Slave 网络协议数据结构（当前未启用）。

## 5.2 Infra 层 (mcm_infra)

提供跨模块的基础设施工具。

- **`logger.rs`**: `InstanceLogger` 实例级日志器，支持按模块名分类、文件输出和 UI 面板显示。
- **`config_loader.rs`**: 统一的 JSON 配置文件加载/保存工具。

## 5.3 Core 层 (mcm_core)

- **`params.rs`**: 
    - `MonitorParams`: 基于 `nih-plug` 的参数树（使用 `#[derive(Params)]`）。
    - 包含 `master_gain`, `dim`, `cut`, `mono`, `low_boost`, `high_boost`, `lfe_add_10db` 等参数。
    - `PluginRole` 枚举: `Standalone` / `Master` / `Slave`。
    - `ChannelParams`: 每通道的 `enable` / `solo` / `mute` 参数（固定 32 通道上限）。
- **`interaction.rs`**:
    - `InteractionManager`: 业务逻辑核心。管理 `Solo/Mute` 组逻辑、`Primary/Compare` 模式切换。
    - **线程安全设计**: 使用 `RwLock` 保护非实时状态集合，`AtomicCell` 提供音频线程的无锁快照读取。
- **`audio.rs`**:
    - `process_audio_with_layout()`: 核心音频处理函数。接受预计算的 `Layout` 引用和 `RenderSnapshot`。
    - 优化特性: 栈上增益缓存、无分支 (Branchless) 计算、通道优先内存连续访问。
- **`osc_state.rs`**: `OscSharedState` — OSC 共享状态管理，支持 pending 标志位的异步参数同步。
- **`config_manager.rs`**: `ConfigManager` — 通道布局配置管理，`O(1)` 通道名称查找表。

## 5.4 Reactor 层 (mcm_reactor)

基于 `tokio` 的 Sidecar 模式。

- **`lib.rs / Reactor`**: 
    - 启动一个后台线程运行 Tokio Runtime。
    - 通过 `crossbeam::channel` 接收宿主指令 (`StartWeb`, `InitOsc`, `BroadcastState`, `Shutdown`)。
- **`actors/web.rs`**:
    - 基于 `Axum` 的 HTTP/WebSocket 服务（动态端口分配）。
    - 提供静态文件服务（`rust-embed` 内嵌前端页面）和 WebSocket API。
    - **当前限制**: Solo/Mute/通道点击已实现；参数修改（Volume/Dim/Cut 等）因 nih-plug 参数可见性限制尚未接入（标记为 TODO）。
    - **当前限制**: `state_push_task` 状态推送逻辑待完成（WebSocket 客户端暂无法接收实时状态更新）。
- **`actors/osc.rs`**:
    - 基于 `rosc` 的 UDP 三线程架构（Send / Receive / Blink）。
    - 维护 `OscSharedState`，实现硬件控制器 ↔ 插件的双向 OSC 同步。
    - 支持 Loupedeck/StreamDeck 等硬件的 LED 状态反馈和 Blink 动画。
- ~~**`actors/network.rs`**~~:
    - ZeroMQ PUB/SUB 实现，负责跨机状态广播。
    - **当前状态**: 已从模块树断开（`mod.rs` 中注释掉），文件仍存在于磁盘，待清理或重新启用。

## 5.5 Plugin 层 (monitor_controller_max)

- **`lib.rs`**: 插件入口，实现 `ClapPlugin` / `Vst3Plugin` trait。管理生命周期、初始化 Reactor、驱动音频处理。
- **`editor.rs`**: 
    - GUI 入口。通过 `gui-slint` feature flag 切换到 Slint 框架。
    - `create_editor()` 函数组装所有共享状态并创建 `SlintHostEditor` 实例。
- **`editor/slint_host.rs`**: Slint GUI 宿主适配层（54KB），负责将 Slint UI 与 nih-plug 编辑器接口对接。
- **`ui/`**: Slint 声明式 UI 定义：
    - `main.slint`: 窗口入口和全局属性/回调绑定。
    - `pages/home_page.slint`: 主界面布局（73KB）。
    - `globals/theme.slint`: 主题颜色和字体定义。
